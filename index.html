[DAX Studio](https://daxstudio.org/) まとめ（未経験者向け）

DAX Studioは、Power BI Desktop 等のタブラー系モデルに接続し、DAXクエリを「書く・実行する・性能を分析する」ためのツール。 まずは Server Timings で計測→改善のループを回すのが最短。
公式: https://daxstudio.org/ / GitHub: https://github.com/DaxStudio/DaxStudio

閲覧: アカウント不要 用途: DAX実行・性能分析 対象: Power BI / Tabular

[DAX Studio](https://daxstudio.org/) とは

・何ができる？
・DAXクエリの作成・実行、実行結果とパフォーマンスの可視化（Server Timings等）
・誰が使う？
・Power BIで「遅い」「重い」を潰したい人、複雑なメジャーの動作確認をしたい人

参考: 公式の概要ページで「Powerful performance tuning tools（Server Timings等）」が紹介されている。 公式サイト

導入（インストール・起動）

ダウンロード

・公式ダウンロード: https://daxstudio.org/downloads/

Power BI Desktopから起動できる理由（外部ツール連携）

・Power BI Desktopの「外部ツール」リボンから起動すると、内部のAnalysis Servicesインスタンス情報がツールに渡り、自動接続できる。
・参考: Microsoft Learn「外部ツール」説明 https://learn.microsoft.com/ja-jp/power-bi/transform-model/desktop-external-tools
Power BI Desktopの外部ツール リボン（Microsoft Learnより） [Source] desktop-external-tools

まず覚える画面（最短で戦力化）

Server Timings（性能計測の基本）

・目的: DAX実行時の時間を取り、遅さの要因を切り分ける
・最初の使い方: Server Timingsを有効化 → 実行 → 数字を見る → 修正 → 再計測
Server Timings（DAX Studio公式ドキュメントより） [Source] server-timings-trace
